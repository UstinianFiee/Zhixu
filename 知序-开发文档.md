# 知序 —— 遣词作序，静揽人间文柔

## 基于 Vue3 + FastAPI 的知序应用开发文档 · V1.0

---

## 一、项目概述

### 1.1 项目定位

**知序** 是一款融合轻文艺内容与英语学习功能的 Web 应用。它参考小独 App"每天一首歌、一段话、一篇文"的文艺气质，结合 Qwerty Learner 的单词记忆与键盘练习，为用户提供 **"遣词作序，静揽人间文柔"** 的独特体验——在文字与语言的秩序中安顿身心，用每日的积累编织属于自己的精神序章。

### 1.2 设计理念

- **遣词作序**：主动驾驭语言，通过每日单词练习、打字默写，为自己的人生书写有序的篇章。
- **静揽人间文柔**：以宁静的姿态，拥抱世间一切温柔的文字、音乐与思想。

### 1.3 目标用户

- 英语学习者（学生、职场人士）
- 键盘工作者与程序员（提升英文输入速度）
- 文艺爱好者（追求精神轻食）
- 追求碎片化高效学习与内心安宁的都市人群

### 1.4 主题色

- 暮蓝 `#2C3E50`（主色调）
- 暖米 `#F5F0E6`（背景色）
- 点缀金 `#D4AF37`（强调色）

## 二、技术架构

### 2.1 前端

| 技术 | 用途 |
|------|------|
| Vue 3 + TypeScript | 核心框架，组合式 API |
| Vite | 构建工具，极速热更新 |
| Pinia | 状态管理 |
| Tailwind CSS | 原子化样式框架 |
| Axios | HTTP 请求 |
| Vue Router | SPA 路由，支持路由守卫 |

### 2.2 后端

| 技术 | 用途 |
|------|------|
| Python 3.12 | 运行环境 |
| FastAPI | Web 框架，自动生成 OpenAPI 文档 |
| SQLModel | ORM（基于 SQLAlchemy + Pydantic） |
| SQLite | 开发环境数据库（零配置） |
| MySQL 8.0 | 生产环境数据库（Docker 部署） |
| PyMySQL | MySQL 驱动 |
| python-jose + bcrypt | JWT 认证 + 密码加密 |

### 2.3 部署

- **开发环境**：本地直接运行 FastAPI + Vite dev server + SQLite
- **生产环境**：Docker Compose 一键部署（前端 Nginx + 后端 Uvicorn + MySQL）

## 四、功能模块

### 4.1 文艺留声 · 每日精选

| 模块 | 说明 |
|------|------|
| 每日一句 | 哲理名言卡片，支持收藏、TTS 朗读 |
| 每日一曲 | 音乐推荐卡片，支持收藏 |
| 每日一文 | 文章阅读卡片，支持收藏、弹窗阅读 |

三张卡片使用统一的 `.daily-card` 样式类，尺寸一致、响应式排列。首页配有 fadeIn / slideUp / cardAppear CSS 动画。

### 4.2 单词簿 · 键盘学习

| 模块 | 说明 |
|------|------|
| 词库列表 | 浏览内置词库（CET-4、CET-6 等） |
| 打字练习 | 逐字符输入，实时绿/红反馈，强制纠错，TTS 发音 |
| 默写模式 | 隐藏拼写，仅显示中文释义，自由输入 |
| 错题本 | 自动收集错误单词，按错误次数排序 |
| 学习看板 | 今日/累计统计：总记录、正确数、错误数、平均速度、积分、等级 |

### 4.3 用户体系

| 功能 | 说明 |
|------|------|
| 注册/登录 | JWT 认证（access + refresh token） |
| 个人中心 | 昵称编辑、头像上传（本地文件或 URL）、积分/等级显示 |
| 每日签到 | 日历视图，连续签到天数，积分奖励（+3 基础，每 7 天 +30） |
| 收藏夹 | 按类型（quote/song/article）展示收藏内容 |

### 4.4 积分系统

| 行为 | 积分 |
|------|------|
| 完成 1 个单词练习 | +2 |
| 连续正确 10 个单词 | +5（额外奖励） |
| 顺利完成一组练习（≥10 词） | +3 |
| 每日签到 | +3（基础） |
| 连续签到 7 天 | +30（额外） |
| 新用户注册 | +50 |

### 4.6 排行榜

| 类型 | 说明 |
|------|------|
| 积分总榜 | 累计积分排名 |
| 本月榜 | 本月积分增量 |

### 4.7 后台管理（管理员）

| 功能 | 说明 |
|------|------|
| 内容管理 | 筛选（全部/句/曲/文）、创建、编辑、删除 |

## 七、Docker 部署

### 7.1 架构

```
浏览器 → :1234 → [frontend: nginx] → 静态文件
                         ↓ proxy /api/
                   :8888 → [backend: uvicorn] → FastAPI
                                                   ↓
                         :3306 → [db: mysql:8.0]
```

### 7.2 一键启动

```bash
docker compose up -d --build
```

| 服务 | 端口 | 说明 |
|------|------|------|
| frontend | 1234 | Nginx 静态文件 + API 反向代理 |
| backend | 8888 | FastAPI + Uvicorn |
| db | 13306 | MySQL 8.0 |

### 7.3 关键配置

- **`docker-compose.yml`**：三个服务编排，MySQL 健康检查，数据卷持久化
- **`backend/entrypoint.sh`**：等待 MySQL 就绪 → 运行种子数据 → 启动 Uvicorn
- **`backend/Dockerfile`**：Python 3.12-slim，种子数据在运行时导入（非构建时）
- **`frontend/nginx.conf`**：监听 1234，`/api/` 代理到 `backend:8888`，SPA `try_files`
- **`backend/app/database.py`**：通过 `DATABASE_URL` 环境变量自动选择 SQLite 或 MySQL

### 7.4 默认账号

| 账号 | 密码 | 角色 |
|------|------|------|
| admin | admin123 | 管理员 |
| demo | demo123 | 普通用户（500 积分） |

## 八、本地开发

### 8.1 后端

```bash
cd backend
pip install -r requirements.txt
python -m app.seed        # 导入种子数据（幂等）
uvicorn app.main:app --reload --port 8000
```

访问 http://localhost:8000/docs 查看 API 文档。

### 8.2 前端

```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:5173，Vite 代理 `/api` 到后端 8000 端口。

### 8.3 数据库

开发环境默认使用 SQLite（`backend/zhixu.db`），零配置，单文件。通过 `DATABASE_URL` 环境变量可切换到 MySQL。
